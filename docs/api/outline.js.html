Outline

Outline

Processes a conversation modelling schema, with attributes for conversation nodes (including scenes, dialogues, paths and directors) defined as bits.

Outlines can be loaded from YAML files or passed directly as an array of objects, to initialise the listeners and callbacks to execute each bit.

Bits can lead to consequitive bits, with branches to follow as responses are matched against the bit’s condition. Those with a listen attribute are effectively global entry points to a scene, where others are only accessesable in the scope of a parent scene.

A subsequent bit can even lead back to it’s own parent or any other bit (global or not) but the scene scope can’t change once entered.

Kind: global class

new Outline(bits, [options], [key])

Param Type Description
bits Array.<string/Object> Attributes to setup bits (accepts YAML file path)
bits[].send array String/s to send when doing bit (minimum requirement)
[bits[].catch] string To send if response unmatched by listeners
[bits[].condition] string Converted to regex for listener to trigger bit
[bits[].listen] string Type of listener (hear respond) for scene entry bit
[bits[].scene] string Scope for the scene (only used if it has a listen type)
[bits[].key] string Key for scene and/or dialogue running the bit
[bits[].options] Object Key/val options for scene and/or dialogue config
[bits[].next] array Key/s (strings) for consequitive bits
[options] Object Key/val options for outline config
[key] string Key name for this instance

outline.doBit(res, key)

Send messages and setup any following listeners for a bit.

Called in an open dialogue, as a callback on entering the scene or continuing from a prior bit. Adds the bit as a property of the response.

Kind: instance method of Outline

Param Type Description
res Response Hubot Response object
key string The key for a loaded bit

outline.setupScene(bit)

Prepare the arguments required to add listener for a scene entering bit.

Only applicable to bits with a listen attribute (hear or respond). Subsequent bits will play out within the same scene so their listen type is irrelevant because all responses from an engaged audience will be routed through the current dialogue.

Bits only require a .listen and .condition property to setup a scene listener and will use defaults if .scope and .options are null.

Kind: instance method of Outline

Param Type Description
bit Object Attributes to setup scene for entry to bit

outline.getSceneArgs() ⇒ array

Get arguments to pass into scene listeners for global bits.

Kind: instance method of Outline
Returns: array - Items contain required arguments (call with spread syntax)
Example (The hard way) 

const scene = new Scene(robot)
const outline = new Outline('./conversation.yml')
outline.getSceneArgs().map((args) => scene.listen(...args)

Example (The easy way) 

const pb = new Playbook(robot).outline('./conversation.yml')
// ^ playbook helper creates outline and sets up scenes
